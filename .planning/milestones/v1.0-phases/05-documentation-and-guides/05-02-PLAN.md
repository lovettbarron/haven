Create docs/troubleshooting-guide.md with dual structure per CONTEXT.md decisions.

**Part 1: Symptom-Based Main Guide**

Organize by what you SEE. Cover these symptom categories (minimum -- add more if relevant to planted crops):

- **Leaves turning yellow** (bottom leaves normal vs whole plant = overwatering/nutrient deficiency)
- **Holes in leaves** (slugs, caterpillars, flea beetles -- identify by hole pattern)
- **White powder on leaves** (powdery mildew -- especially cucumbers, kale)
- **Plant wilting** (underwatering vs root rot vs heat stress)
- **Spots on leaves** (fungal disease, nutrient deficiency)
- **Aphids on plant** (FEATURED ENTRY: "watch and wait for ladybugs" philosophy from Phase 3 -- do NOT spray, borage/nasturtium are planted as attractors for predators)
- **No fruit / flowers dropping** (pollination issues, temperature stress)
- **Seedlings falling over** (damping off, slug damage)
- **Stunted growth** (cold soil, nutrient lock, root-bound)

Each symptom entry MUST follow this format:
```
## [Symptom Name]

### [Specific variant]
**Severity:** [emoji] [timeframe label]
- Green: No rush (cosmetic/minor)
- Yellow: This week (needs attention soon)
- Red: Today (plant at risk)

**What's happening:** [Explanation in plain language]
**Affects:** [Which crops in THIS garden are susceptible]

**What to do:** [Step-by-step action for father]
**Child action:** "[Concrete task the child can do]"

[PHOTO: description of what to capture]
```

The aphid entry must be prominent (not buried) and explicitly state: observe for 3-5 days, check for ladybug larvae before any action, nasturtiums serve as trap crops by design, borage attracts predatory hoverflies.
